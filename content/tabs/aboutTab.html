CCDump - Cycle Collector Heap Dump

Display the graph created by cycle collector - use when hunting down memory leaks.

Author: Jan Odvarko, odvarko@gmail.com


Tips

See some tips about how to use this extension.

  1. You mostly want to start by clicking on Run CC Analysys button in the Home tab. After you do it, Firefox cycle collector runs three times to clean up garbage and consequently once, with a listener, to collect entire CC heap graph of objects in the memory. The result object graph is immediatelly displayed as a table where each entry represents an object.

  2. The prefered way how to explore the graph is to search it. Use a search box located at the top right corner of the Home page. Note that if there is a string in the box the search runs automatically just after analysys.

  3. You will typically look for documents that shouldn't be in the memory since the parent tab has been already closed (so called zombie documents). Just search for nsDocument or pick Find Zombie Documents from the search option menu. You can also search for http to get all zombie elements.

  4. You can search for any other keywords within object's name or use object's address. You can also use regular expressions.

  5. As soon as you have some search results, e.g. zombie documents, you can continue exploring object graph related to the selected document. Click the small black triangle next to object's name and pick one of the options.

    • Show Details - See detailed information about the object.
    • Show Roots - See root objects that keeps this object in the memory.
    • Show Graph - See entire graph of all related objets as a expandable tree.

  6. Especially Show Graph allows to dramatically reduce the object graph to see only related objects and search further.

  7. Note that this is a bootstrapped extension and so, you can install it any time when you experience a memory leak in your running browser and immediatelly start exploring your current CC graph (without browser restart).



Object Fields

You can see following fileds when dealing with objects in the graph.
  • name - objects name generated by CC collector
  • address - unique hexadecimal address of the object
  • refcount - number of objects referencing the object
  • gcmarked - if false CC is OK with freeing the object
  • root - if true the object is a root
  • edges - array of objects referenced by this object
  • owners - array of parent objects referencing this object


API

The extension is build on top of nsICycleCollectorListener interface introduced in Firefox 13.
